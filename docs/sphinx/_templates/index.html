Welcome to StarCluster main documentation. StarCluster is a no-nonsense packaging tool solution for python softwares, targeted as an alternative to distutils/setuptools/distribute/etc.... StarCluster aims at being simple, hackable, and extensible - in other words, pythonic !

StarCluster is under constant development. The main implemented features are

  • Declarative description of your package. Metadata (name, version, etc...) as well as data files, options and package content are described in a simple format.
  • Flexible installation scheme: although the installer inluded with StarCluster use sensible platform-dependent defaults, you can install any file anywhere.
  • Simple data files handling: installing configuration files, library files, etc... at the right place is now possible. The install paths do not need to be hardcoded, and may be customized at will through StarCluster command line interface
  • Layered internal architecture: packaging, command line interface, are clearly separated.
  • Hackable and extensible: StarCluster is designed from the ground up to be extensible. It includes its own small build tool for C extensions, but it is possible to use something else like waf, scons or even distutils (for compatibility reason).
  • Scale down: a core principle of StarCluster is to make simple softwares easier to package, and avoid too many choices.
  • Scale up: on the other hand, thanks to extensive extensibility, StarCluster can be used for complex packages. StarCluster is already able to build numpy and scipy, with much less boilerplate than with the current numpy distutils extensions.
